Final Project 1: Shift-and-Add Multiplier

This project is a continuation from a previous course. Your task is to enhance this multiplier, making it function as an 8-bit multiplier to generate a 16-bit multiplication result. Follow the steps below:

  1. Implement each block as an independent Verilog module. There are four functional blocks: two shift registers, one adder, and one register. Additionally, there's a controller (FSM).
  1. Construct a main module named "datapath" to encompass all blocks, excluding the controller.
  1. Subsequently, create a module named "mul8" to combine the datapath module and the controller (FSM) module.
  1. When the Load signal is set to 1, "mul8" will load the two operands (a[7:0] and b[7:0]) into the shift registers.
  1. As the Load signal transitions to 0, the left shift register performs a shift-to-left operation on the positive edge of the clk signal, while the right shift register executes a shift-to-right operation.
  1. The result is stored in the register, capturing the addition result from the adder when the write signal is set to 1.
  1. The controller (FSM) must coordinate all the blocks to work together, executing a simple multiplication algorithm based on shift-and-add.
  1. Additionally, you need to write a testbench for each block and the top-level module (mul8) to ensure all the sub-modules are working correctly.

module mul8 (
    input [7:0] a,
    output [7:0] b,
    input rst,
    input clk,
    input load
);

	wire shr_ld, shl_ld, write;
  
    // Instantiate datapath
    datapath dp (
				.clk(clk),
        .a(a),
        .b(b),
				.shr_ld(shr_ld),
				.shl_ld(shl_ld),
				.write(write)
    );
    
    // Instantiate Controller (FSM)
    controller fsm (
        .rst(rst),
        .clk(clk),
        .load(load)
				.shr_ld(shr_ld),
				.shl_ld(shl_ld),
				.write(write)
    );

endmodule

Objective: This project aims to expand a 4-bit multiplier into an 8-bit multiplier. The approach involves implementing a multi-module design using Verilog.

Tasks:

  1. Modular Design:
    • Functional Blocks: Implement four independent Verilog modules:
      • Two Shift Registers: One for each operand.
      • One Adder: Performs partial product accumulation.
      • One Register: Stores the final multiplication result.
    • Controller (FSM): Coordinates the operation of all blocks and executes the shift-and-add multiplication algorithm.
  1. Data Path and Top-Level Module:
    • Datapath Module (datapath): Combines all functional blocks except the controller.
    • Top-Level Module (mul8): Combines the datapath and controller modules.
  1. Operation Flow:
    • Load Signal: When set to 1, mul8 loads operands a[7:0] and b[7:0] into the shift registers.
    • Multiplication Cycle:
      • On Load transition to 0:
        • Left Shift Register: Shifts operand a left on positive clk edges.
        • Right Shift Register: Shifts operand b right on positive clk edges.
        • Adder: Accumulates partial products based on rightmost bits of both registers.
      • Write Signal: When set to 1, the final sum from the adder is stored in the register.
  1. Testing:
    • Write testbenches for each individual block (shift registers, adder, register, controller) and the top-level module (mul8) to verify their functionality.

Additional Notes:

  • Provide detailed specifications for each module, including input/output signals and functionality.
  • Implement a clear and efficient FSM design for the controller.
  • Pay attention to proper data flow and register usage to ensure accurate multiplication results.
  • Use appropriate Verilog coding practices for modularity, readability, and maintainability.
