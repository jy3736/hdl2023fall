Combinational Logics Using Verilog

Introduction to Combinational Logic

Combinational logic is a type of digital logic circuit where the output depends only on the current value of the inputs. This is in contrast to sequential logic circuits, where the output depends on both the current input values and the previous state of the circuit.

Combinational logic circuits are often used to implement simple operations, such as arithmetic, boolean algebra, and data processing. For example, a combinational logic circuit could be used to implement a full adder, which is a circuit that can add two numbers together.

Verilog

Verilog is a hardware description language (HDL) that is used to describe digital logic circuits. HDL languages are used to describe the functionality of a circuit, rather than its physical implementation. This makes them ideal for designing complex circuits, as the designer can focus on the functionality of the circuit without having to worry about the details of how it will be implemented.

To describe a combinational logic circuit in Verilog, you first need to define the inputs and outputs of the circuit. This is done using the module keyword. For example, the following Verilog code defines a module called adder with two inputs (a and b) and one output (sum):

module adder(
    input [31:0] a,
    input [31:0] b,
    output [31:0] sum
);
endmodule

Once the inputs and outputs of the circuit have been defined, you can then describe the functionality of the circuit using the assign keyword. The assign keyword is used to assign a value to an output signal. For example, the following Verilog code assigns the sum of a and b to the output signal sum:

module adder(
    input [31:0] a,
    input [31:0] b,
    output [31:0] sum
);

assign sum = a + b;

endmodule

Example: Adder

The following Verilog code shows a complete Verilog module for an adder:

module adder(
    input [31:0] a,
    input [31:0] b,
    output [31:0] sum
);

assign sum = a + b;

endmodule

This module can be used to instantiate an adder circuit in a larger design. For example, the following Verilog code instantiates an adder circuit and connects its inputs and outputs to other signals in the design:

module top(
    input [31:0] a,
    input [31:0] b,
    output [31:0] sum
);

adder adder_inst(
    .a(a),
    .b(b),
    .sum(sum)
);

endmodule

Exercises

  1. Write a Verilog module for a 2-to-1 multiplexer. The module should have three inputs (a, b, and sel) and one output (out). The sel signal should select which of the inputs a or b is routed to the output out.
  1. Write a Verilog module for a 4-bit comparator. The module should have two 4-bit inputs (a and b) and three outputs (lt, eq, and gt). The lt output should be asserted if a is less than b. The eq output should be asserted if a is equal to b. The gt output should be asserted if a is greater than b.
  1. Write a Verilog module for a parity generator. The module should have one input (in) and one output (out). The out signal should assert if the number of 1 bits in the input in is even.
  1. Write a Verilog module for a BCD to decimal converter. The module should have one 4-bit input (bcd) and one 4-bit output (dec). The dec output should be the decimal representation of the BCD input bcd.
  1. Write a Verilog module for a 32-bit multiplier. The module should have two 32-bit inputs (a and b) and one 64-bit output (product). The product output should be the product of the two inputs a and b.
