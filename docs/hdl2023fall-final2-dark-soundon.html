Final Project 2: SPI Data Communication Setup

Please consult the depicted SPI data communication arrangement. Construct two distinct 8-bit SPI modules, one serving as the master and the other as the slave. Develop each module autonomously and conduct thorough full-coverage tests using a testbench. Subsequently, assemble them into a unified module named "spi8" according to the figure. Afterward, devise a testbench to furnish randomized data for validating the SPI data communication setup implemented by the "spi8" module. The specifics of the SPI Master module (spimaster) and the SPI Slave module (spislave) are elucidated below.

SPI Master:

Interface:

  • dinM[7:0]: This is where you input an 8-bit data that the master wants to send to the slave.
  • rst, clk, transfer, loadM: These are control signals.
    • rst: Resets the system.
    • clk: The clock signal, like a heartbeat, making everything happen in sync.
    • transfer: Signals when the master wants to start sending data.
    • loadM: Signals when new data should be loaded into the master's internal storage.
  • miso: This is where the master receives data from the slave.
  • sclk, mosi, ss: These are output signals.
    • sclk: The clock signal sent by the master to synchronize data transfer.
    • mosi: The output where the master sends data to the slave.
    • ss: Stands for Slave Select. When this is high (1), it means the master is communicating with the slave.
  • masterR[7:0]: This is the output displaying the data in the master's internal storage.

Behavior:

  • Data Loading (loadM): When loadM is 1, the data from dinM is stored in the master's internal register.
  • Data Transfer (transfer): If transfer is 1, and ss is high, the master starts sending 8 pulses (bits) through sclk, synchronized with clk.
  • Clock Synchronization: The rising edge of transfer triggers sclk, and after 8 pulses, sclk pauses at 0.
  • Priority: If transfer is happening, loadM is ignored.
  • Data Output (mosi): The data stored in the internal register is sent out bit by bit through mosi.
  • Display (masterR): masterR always shows what's in the internal register.

SPI Slave:

Interface:

  • dinS[7:0]: This is where you input an 8-bit data that the slave wants to send to the master.
  • rst, clk, loadS: These are control signals.
    • rst: Resets the system.
    • clk: The clock signal, ensuring everything happens in sync.
    • loadS: Signals when new data should be loaded into the slave's internal storage.
  • sclk, mosi, ss: These are input signals from the master.
    • sclk: The clock signal sent by the master to synchronize data transfer.
    • mosi: The input where the slave receives data from the master.
    • ss: Stands for Slave Select. When this is high (1), it means the master is communicating with the slave.
  • miso: This is the output where the slave sends data to the master.
  • slaveR[7:0]: This is the output displaying the data in the slave's internal storage.

Behavior:

  • Data Loading (loadS): When loadS is 1, the data from dinS is stored in the slave's internal register.
  • Data Output (miso): If ss is high, the slave sends 8 pulses (bits) of data to the master, synchronized with sclk.
  • Display (slaveR): slaveR always shows what's in the internal register of the slave.
